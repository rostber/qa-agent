# Playwright CLI agent

Интерактивная CLI-утилита для общения с AI-агентом с поддержкой MCP сервера Playwright.

## Быстрый старт

```bash
# Установка
npm install
npx playwright install

# Настройка (создайте .env файл)
cp .env.example .env

# Запуск
npm start
```

## Конфигурация

Создайте файл `.env` с параметрами:

```env
OPENAI_API_TOKEN=your_token
OPENAI_BASE_URL=http://localhost:11434/v1
OPENAI_MODEL=qwen
```

## Функционал

### Управление браузером

| Команда | Описание |
|---------|----------|
| `/open <url>` | Открыть браузер по адресу |
| `/auto-open` | Открыть последнюю сохранённую URL |
| `/prompt <текст>` | Изменить системный промт агента |
| `/exit` или `/quit` | Выйти из программы |

### Автоматическое сохранение URL

- После команды `/open <url>` адрес автоматически сохраняется в `.qa-agent-state.json`
- При следующем запуске браузер автоматически откроет сохранённую страницу
- Используйте `/auto-open` для ручного открытия сохранённой URL

### Ввод сообщений

- **Однострочный ввод:** введите текст и нажмите Enter
- **Многострочный ввод:** нажмите Enter после каждой строки, пустая строка отправляет сообщение

### Работа агента

1. Откройте браузер командой `/open <url>`
2. Выполните авторизацию вручную в открывшемся окне
3. Давайте агенту задачи на естественном языке

**Важно**: Браузерная сессия сохраняется между запросами. После авторизации вы можете закрывать и открывать приложение — авторизация останется активной.

## Доступные инструменты

Агент может использовать следующие инструменты Playwright MCP:

| Инструмент | Описание |
|------------|----------|
| `browser_navigate` | Перейти по URL |
| `browser_click` | Кликнуть по элементу (требуется `ref` из snapshot) |
| `browser_type` | Ввести текст в поле |
| `browser_fill_form` | Заполнить форму несколькими полями |
| `browser_snapshot` | Получить доступность страницы |
| `browser_take_screenshot` | Сделать скриншот |
| `browser_wait_for` | Ожидать появления текста |
| `browser_press_key` | Нажать клавишу |
| `browser_hover` | Навести курсор |
| `browser_select_option` | Выбрать опцию в выпадающем списке |
| `browser_drag` | Перетащить элемент |

## Поведение агента

- **Последовательное выполнение:** агент выполняет шаги один за другим
- **Проверка ошибок:** если шаг завершается ошибкой, выполнение прекращается
- **Отчёт о результате:** после выполнения всех шагов агент выводит итоговый отчёт
- **Сохранение сессии:** браузерная сессия (cookies, localStorage) сохраняется между запросами и запусками приложения

## Важные замечания

### Использование `ref` для кликов

Инструмент `browser_click` требует параметр `ref` — уникальный идентификатор элемента из snapshot.

**Пример snapshot:**
```yaml
- heading "Example Domain" [ref=e3]
- link "Learn more" [ref=e6]
```

**Правильное использование:**
```
browser_click: {"ref": "e6", "element": "Learn more link"}
```

**Неправильное использование:**
```
browser_click: {"ref": "Learn more"}  # ❌ Ошибка: ref должен быть идентификатором, а не текстом
```

### Цветной вывод

Приложение использует цветовое оформление для улучшения читаемости:

- 🟢 **Зелёный** — успешные операции
- 🔴 **Красный** — ошибки
- 🟡 **Жёлтый** — предупреждения
- 🔵 **Синий** — информация
- ⚪ **Серый** — приглушённый текст

│   ├── agent.ts           # Логика AI-агента
